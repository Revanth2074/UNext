# NimbusTech Support Ticket Triage & Escalation API - Module 1

This module contains the ticket processing command-line script which validates raw nightly ticket CSV exports, identifies SLA breaches, scores ticket priorities, and generates a structured JSON report.

## Directory Structure

```
day13/
├── data/
│   ├── tickets_raw.csv          # Sample raw tickets CSV (corrupted/invalid rows)
│   ├── tickets_valid.csv        # Valid tickets CSV for testing successful runs
│   └── tickets_report.json      # Structured JSON report generated by processor
└── ticket_processor/
    ├── __init__.py
    ├── config.py                # Configuration constants
    ├── models.py                # Dataclasses for ticket structures
    ├── validators.py            # CSV row and column validation
    ├── processor.py             # SLA evaluation & statistics compilation
    └── main.py                  # CLI command runner & logging configuration
```

## How to Run

Ensure you are in the project root directory (`day13/`).

### 1. Process Valid Tickets CSV
To generate a JSON report from a valid dataset:
```bash
python -m ticket_processor.main --input data/tickets_valid.csv --output data/tickets_report.json
```

### 2. Process Raw CSV (With Corrupted Rows > 10%)
To test the abort-on-bad-data logic (the script will log an error and exit with a non-zero status code without writing the JSON report):
```bash
python -m ticket_processor.main --input data/tickets_raw.csv --output data/tickets_report.json
```
