# 🎫 NimbusTech Support Ticket Triage & Escalation Service

This repository contains the MVP implementation of the NimbusTech Support Ticket Triage & Escalation system, comprising two primary modules:
1. **Module 1: Ticket Processing Script** - A command-line script that processes, validates, and analyzes nightly CSV ticket exports, generating a clean JSON report.
2. **Module 2: FastAPI Ticket API** - A lightweight internal API that loads the processed JSON report at startup and exposes CRUD operations over HTTP.

---

## ⚙️ Setup and Installation

1. Navigate to the root directory `nimbustech_ticket_api/`.
2. Install the dependencies listed in `requirements.txt`:
   ```bash
   pip install -r requirements.txt
   ```

---

## 🚀 How to Run

### 1. Module 1: Ticket Processor (CLI)

Run the script from the root directory using python module execution syntax:

* **To process clean data successfully**:
  ```bash
  python -m ticket_processor.main --input data/tickets_valid.csv --output data/tickets_report.json
  ```
* **To verify the automatic abort threshold (>10% invalid rows)**:
  ```bash
  python -m ticket_processor.main --input data/tickets_raw.csv --output data/tickets_report.json
  ```
  *(The script will log an error and exit with a non-zero exit code without generating a report, as designed).*

### 2. Module 2: FastAPI Service

Start the uvicorn development server:
```bash
uvicorn ticket_api.main:app --reload
```

The service will start at **`http://127.0.0.1:8000`**.
Open the interactive Swagger documentation at **`http://127.0.0.1:8000/docs`** to test endpoints manually.

## 🔌 API Documentation

### Core Endpoints

* **`GET /`** - Basic root service status.
* **`GET /tickets`** - Returns all tickets in the store, sorted by `priority_score` (descending).
* **`GET /tickets/breached`** - Returns only tickets whose SLA has breached (`created_at + sla_hours < now` and status is not `closed`).
* **`GET /tickets/{ticket_id}`** - Fetches a ticket by ID. Returns `404 Not Found` if missing.
* **`POST /tickets`** - Creates a new ticket. Validates input fields and calculates `priority_score` and `sla_breached`.
* **`PUT /tickets/{ticket_id}`** - Updates an existing ticket's `status` and/or `priority_raw`.
* **`DELETE /tickets/{ticket_id}`** - Removes a ticket from the in-memory store.

### Stretch Goal Endpoints

* **`GET /tickets/summary`** - Returns a live counts breakdown (total, breached, and counts by category).
* **`GET /tickets/category/{category}`** - Filters tickets by category.
* **`PATCH /tickets/{ticket_id}`** - Perform partial updates of `status` and/or `priority_raw`.
